网站特点

                
  1. 我们对参与网站心理健康测试的用户短时间内给予心理健康分析以及方向引导
    2. 
            
  2. 将得到的数据按用户意愿选择性保留到网站数据库中
    3. 
            
  3. 此类数据将用于定期分析大学生心理健康问题现状的趋势分析问题，以及为更多心理研究机构提供更加具有时效性的、直接的研究材料和数据。
    4.

设计任务

                
  1. 进行心理测试和心理健康指南相关资料的收集，完成网页数据库的建立
    2. 
            
  2. 设计网页，划分区域，建立各个区域的模块
    3. 
            
  3. 确定网页各个区域模块的功能，为网页添加符合大众审美的视觉设计，为增强用户体验感加入便捷的交互设计
    4. 
            
  4. 对网页进行BUG测试，并实现网页的上线。要求交互设计方便用户使用，视觉设计符合审美，各模块功能分明划分整齐，心理测试以及指导资料具有可信性和可行性。
    5.

问题与解决

    
            
  • 技术方面，系统所需理论知识欠缺，本系统需要实现的一些功能以小组现有技术
                可能难以实现————采取分工，每个成员仅需了解并熟悉分内工作所需的技术及能
                力，如此既能节省设计时间，也能强化各小组成员的职能，提高大家的网站搭建
                和统筹能力；
            
    • 
            
  • 操作可行性方面，用户能否根据指引正确地使用该心理测试网页————尽量使我们
                的网站制作更加明显简洁但又功能齐全，获取用户的授权后才会保留数据为今后
                研究所用。我们的流程也将展示地十分清晰明了，对用户的操作做到步步引导乃
                至网站使用指南。
            
    • 
            
  • 数据方面，系统设计所需数据获取渠道多样且复杂，因此存在数据能否实现稳定
                收集、收集所得数据是否可靠以及如何正确分析所得数据等问题————前期大量投
                放各种心里调查问卷并定向回复心理研究反馈，逐步增大网站知名度、曝光率、
                影响力，不断宣传我们的网站，向大学生等受众群体开放通道、网站。将经用户
                允许所保留到数据库中的数据进行分类计算等处理，与各种心理学理论对比，从
                而保证所得数据的开发利用最大化，不断细化我们的问卷、答题等方面，从而保
                证数据的可靠性和实用性。
            
    • 
        
    





    
预期结果


1，网站可以按照时间安排完成各个区域模块的功能设计与后期修复、维护，顺利完成网站上线 。


2，上线行后稳定运行，测试结果较准确，用户体验感较好，能够顺利实现网站设计的价值，为用户完成个人心理问题的初步检查并提供相应的健康建议。


3，在用户许可的条件下完成问题数据记录，并在不泄露用户隐私的前提下为心理健康课题研究提供相应的数据支持。
